# Security Policy

## Supported Versions

Only the latest version of Paisa.Log is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Paisa.Log seriously. If you discover a security vulnerability within this project, please follow the steps below:

1.  **Do not disclose it publicly** until it is resolved.
2.  Send a detailed vulnerability report to the primary maintainer via GitHub issues (marked as private if possible) or the contact method listed in the project description.
3.  We will acknowledge your report within 48 hours and provide a timeline for a fix.

### Zero-Knowledge Scope
Note that Paisa.Log is designed so that the database administrators have no access to your encrypted data. If you find a way to bypass the local encryption or derive keys without the PIN, please report it immediately as a critical vulnerability.
